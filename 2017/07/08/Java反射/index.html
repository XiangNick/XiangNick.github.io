Java反射

2017-07-08

一、类的初始化过程

加载->验证->准备->解析->初始化->使用->卸载
其中 验证、准备、解析的过程叫做连接。

当程序主动使用到某个类时，如果该类还没被加载进内存，则系统会通过加载、连接、初始化三个步骤来对该类进行初始化

1. 类的加载

类加载时指将该类的class文件加载入内存中，并为之创建一个java.lang.Class对象，我们称之为字节码对象。即类一旦被加载进内存，就会变成字节码对象

类的加载过程由类加载器完成，类加载器通常由JVM提供，我们称之为系统类加载器，我们也可以继承ClassLoader类来提供不同的类加载器可以实现加载本地字节码文件，jar包中的字节码，通过网络加载字节码等。

2. 类的连接

当类被加载进内存后，系统为之生产一个对应的Class对象，接着把类的二进制数据合并到JRE中。

  1. 验证：

    检测被加载的类是否有正确的数据结构

  2. 准备

    负责为类的static变量分配内存，并设置默认值

  3. 解析

    把类的二进制数据中的符号引用替换为直接引用

    直接引用：直接引用某块内存空间
    符号引用：通过一些别名来引用内存空间
    比如把com.xiangnick.website.Customer.username这个符号引用指向于内存

3. 类的初始化

在此阶段，JVM负责对类进行初始化，主要就是对static变量进行初始化

类的初始化：一个类包含以下几个步骤

  1. 如果该类还未被加载和连接，则程序先加载并连接该类

  2. 如果该类的直接父类还未被初始化，则优先初始化其父类。

  3. 如果类中有初始化语句(静态代码块)，则系统依次执行这些初始化语句。

二、什么是反射

1.编译类型、运行类型

  • 编译类型： Object

  • 运行类型： java.util.Date

需求：通过obj对象，调用java.util.Date类中的toLocaleString方法。

此时编译报错，编译时，会检查该编译类型中是否存在该方法，如果存在，编译成功，否则编译失败。

解决方案：因为obj的真实类型是java.util.Date类型，所以我们可以强转，这样才能调用成功。

如果不知道obj的真实类型，就不能强转

如何解决？

2.反射

类用于描述对象，java.lang.Class用于描述类

元数据(metadata):描述数据的描述数据。

反射：就是获取类的元数据，即获取目标类的Class对象的过程。在运行时期，动态地去获取某一个类中的成员信息(构造器、方法、字段、内部类、接口、父类、包等等)。
并且把类中的每一种成员，都描述成一个如下新的类。

  • Class : 表示所有的类
  • Constructor ： 表示所有的构造器
  • Method ： 表示所有的方法

  • Field ： 表示所有的字段

    理解：一个Person对象有性别，年龄，姓名的字段，则用于描述Person对象的Person类有可以获取Person对象的性别，年龄，姓名的方法。
    同理，所有的类都可以有构造器、方法、字段、内部类、父类、接口、包，则用于描述类的Class类有可以获取类的构造器、方法、字段、内部类、父类、接口、包的方法。

    eg: Eclipse的OutLine大纲罗列了我们当前写的class类的属性、字段等，这就是用反射写的。

    三、Class类和Class实例

1. Class类

用来描述类或者接口的类型，即描述类的类。

2. Class类的实例

表示正在运行的Java应用程序中的类和接口。
(即在JVM中已经被加载的类，都是Class类的实例)

每一个class字节码文件被加载入内存时，JVM就会创建(Class的构造器私有化了，只允许JVM创建)一个Class实例来描述该类。

tips: 枚举是一种特殊的类，注解是一种特殊的接口

eg: 当程序第一次使用某一个类，如java.util.Date类的时候，就会把该类的字节码.class文件加载进JVM，并创建出一个Class对象。

问题：Class类用于描述一切类/接口，那Class类的实例到底表示的是哪一份的字节码呢？为了明确区分出Class实例表示的是谁的字节码，Class类提供了泛型。

通过泛型来区分出class实例clazz1和clazz2到底表示的是哪个类的字节码。

3. 如何得到Class的实例

  1. 使用属性，类型.class(就是一份字节码)

  2. Class.forName(String className);
    [框架中使用最多]
    根据一个类的全限定名来构建Class对象
    返回值为Class<?>，会抛出ClassNotFound异常(怕我们乱写全限定名)

  3. 每一个对象都有getClass()方法

    通过实例化的对象的getClass()方法，返回值为Class<?>

    因为实例化对象的getClass()方法和Class的forName()方法返回值都是Class<?>，使用了通配符，相当于没写，相当于是Class<Object>，而我们用Class<java.util.Date>来接收，就是大泛型转小泛型，不强转就会报错。所以我们使用getClass()方法需要强转。

同一个类，在JVM中只存在一份字节码对象，也就是说，上述三种方式获取的Class实例，都是同一个对象

创建对象的步骤

  • 找到构造器所在类的字节码对象
  • 获取构造器对象
  • (设置访问权限)使用反射，创建对象

反射是很暴力的，连私有化的构造器都能拿来在外部构造对象。对于单例模式，私有化构造器也不是那么安全，但是如果单例工厂使用的是枚举，那么反射也无法去创建对象。

2. 调用静态方法

public Object invoke(Object obj,Object… args)

参数：

  • obj:表示被调用方法的所属对象,即用哪个实例对象来调用该方法
  • args:表示调用方法需要的实参

invoke方法第一个参数是需要传递调用该方法的对象的。
但是一些方法是不用对象调用的，比如静态方法，该如何调用呢？

静态方法不属于任何对象，属于类本身。

如果方法是静态的，不论我们第一个obj参数传递的是何值，invoke都不会调用。所以索性直接给null即可，即不需要对象。

3. 调用数组参数方法

3.1 数组元素为基本类型

注：int… arr就是int[] arr，底层还是转成了int[] arr

3.2 数组元素为引用类型

如果数组元素为引用类型，需要在外层再包个new Object[]{}，因为invoke方法要的形参就是Object... args,对于外层包的Object数组，他会自动解包。

使用反射调用方法，在invoke方法中，第二个参数实际参数，统统作为Object数组的元素即可。

即：

通用格式：

❤Method的实例对象.invoke(所属对象,new Object[]{ 所有实参 })❤

如果实参是数组，实参还要再包一层数组。

不论是那种方式的invoke，都可以使用该格式调用。 [ 引用类型的数组参数一定要Object数组包裹 ]

2. Array类

Array类提供了动态创建和访问Java数组的方法。
主要就2种方法，get和set方法

  • get(Object array,int index)
  • set(Object array,int index,Object value)
  • getXX、setXX [ 八种数据类型 ]

对于普通的数组，我们可以通过索引来读取或赋值

但是如果用Object表示的数组，就无法通过索引来操作了，因为编译类型是Object，Object无数组特性，此时就可以用Array的get和set方法了。

练习：使用Array类和反射获取Class实例来实现任意对象数组拷贝方法。
